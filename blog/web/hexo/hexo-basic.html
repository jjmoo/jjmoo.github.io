Hexo小记 —— 新手上路

webhexo
  • hexo
  • web

看到网上说很多牛逼的人都在用 hexo 写博客，我还不牛逼，但是我相信牛逼的人。
Hexo的安装过程就不写了，网上太多。
对前端一窍不通，写这些只是为了记录一下自己的折腾经历。

版本信息

Hexo版本：3.2.0
hexo/package.json如下：

基本配置

修改hexo/_config.yml文件，大部分保持默认就好,
主要就是修改一下站点信息、时间格式什么的，这些也不写了。

文章永久链接地址

默认参数是:

permalink: :year/:month/:day/:title

对应到实际链接就长成了这样：
http://www.jjmoo.com/2016/05/09/folder.../title/
这也意味着蛋疼的目录结构，不管搜索引擎是否喜欢它，我反正不喜欢。
直接改成了：

permalink: blog/:title.html

我的hexo/source/_posts软链接到了一个专门放md文件的目录,
按照文章分类，这个目录下本身是有目录结构的，所以生成的链接就是：
http://www.jjmoo.com/blog/web/hexo/hexo-basic.html
之所以多了一个blog文件夹，是不想网站的根目录下出现太多的内容目录，
而且这样的话，即使以后这个域名下想做点别的事情，也不必影响文章的永久地址了。
其他的配置好像也没什么好讲究的了。

主题

可以在 官方主题 里面挑选，也可以去网上找。
我喜欢的主题是 yilia ，无关风月，就是觉得自己不懂前端，这款皮肤看起来简约美观，相信作者的实力，觉得bug应该会少。
如果图个轻松快活的话，自然是直接git clone到hexo/themes/yilia，跟着作者一起更新，然后再在hexo/_config.yml里面配置：

theme: yilia

不过我还是喜欢多折腾一下，所以fork了一个 仓库 给自己玩。
毕竟看了一天git的使用教程，装模作样整了三个分支：

mine: 平时的所有修改都合入到这个分支
master: 从mine中合入不含个人配置或个人信息的commits
fork: 跟原作者保持一致，并从master中合入想要pull request的commits

主题配置

主题的配置文件也是_config.yml，位于主题的根目录下。
yilia需要修改的配置不多，都非常容易理解，就不一一写了，唯一要留心的就是duoshuo的id修改，把true改成自己的duoshuo id即可。

修改日期图标

日历前面有个日历的图标，上面那个“27”让强迫症寝食难安，所有是一定要改掉的。

对前端一窍不通，所以只能从html、css、js里面找图标的链接，找了半天没找到，后来才知道那图标根本不是图片，而是awesome font图标字体。
第一次接触，觉得简直是黑科技。
在 图标设计引用 里面找到了日历图标的值为0xf073，去修改主题的时候发现作者就是用的这个值，果断替换下载最新版的awesome字体替换source/css/font下的文件，问题解决，神清气爽。

修改全站footer信息

修改layout/_partial/footer.ejs，主要是添加一下备案信息什么的，这个文件很容易看懂，就不多写了。
然后在source/css/_partial/footer.styl中修改底部信息的链接样式，追加如下内容：

#footer a:link,

#footer a:visited,

#footer a:hover,

#footer a:active {
    color:#000000;
    opacity: 0.6;
}

这样的话，链接就看起来不明显了，嘿嘿。

修改标签和分类图标的顺序

作者选择的是先显示标签再显示分类，效果如下：

事实上一般category的数目是固定的，而tag的数目却很不确定，所以把category的图标显示在前应该比较美观。
对调layout/_partial/article.ejs中的17、18行和37、38行即可：

<%- partial(‘post/category’) %>
<%- partial(‘post/tag’) %>

修改post.md

默认的scaffolds/post.md比较简单，每次都要额外添加信息，所以修改了一下：

-–
title: Hexo小记 —— 新手上路
date: 1462765357000
author: Zohn
toc: true
categories:
    - category1
    - category2
tags:
    - tag1
    - tag2
-–

总结

到这里，hexo的基本配置就差不多了。
可以把hexo当工具用来写博客，也可以把它当物料来学习node.js，折腾过程中也了解了一点web的知识，还熟悉了git的使用，个人认为还是很不错的。
