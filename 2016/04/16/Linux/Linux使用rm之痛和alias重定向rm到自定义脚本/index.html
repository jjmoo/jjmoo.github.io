Linux使用rm之痛和alias重定向rm到自定义脚本

今天脑袋有点蒙，做了一件愚蠢的事情。
把前段时间写的代码rm了，用testdisk恢复出来了几十个G的文件，没有文件名和目录结构，还是果断放弃了。
事情是这样的，本来只是想重命名文件夹，腾出名字来svn co的，结果脑袋进水把mv写成了rm，提示是文件夹以后还条件反射的加了个-r参数，于是就彻底悲剧了。
在寻找恢复办法的过程中，看到网上有很多人也遇到了这样的问题，还有些公司甚至禁止使用rm命令。
痛定思痛，为了防止悲剧重演，果断使用alias将rm指向了自己写的shell脚本，顺便温习一下shell脚本的编写。

大致思路如下：
运行rm命令时，执行脚本，屏蔽rm的选项参数，将要rm的文件或文件夹mv到某一个隐藏的临时文件夹中。每天第一次执行这个脚本后，自动清除30天前的文件，以防占用过多的空间。

shell脚本/home/pz/.rm.sh如下：

#!/bin/bash  

# 设置回收路径  
trashPath=$HOME/.MyTrash  

# 设置保存时间,2592000为30天  
period=2592000  

# 创建回收文件夹  
if [ ! -d "$trashPath" ]; then  
    mkdir $trashPath  
fi  

# 清除命令中的所有开关选项  
args=${@##-*}  

# 根据当前时间新建临时文件夹  
dest=$trashPath/`date "+%Y-%m-%d_%H:%M:%S_%s"`/  
if [ ! -d "$dest" ]; then  
    mkdir $dest  
fi  

# 移动文件，如有必要，修改错误信息并输出  
msg=`mv $args "$dest" 2>&1`  
if [ ! "$msg" = "" ]; then  
    echo ${msg//mv/my_rm}  # 将错误信息中的“mv”替换成"my_rm"  
fi  

# =====================================================  
# 每天第一次使用这个命令后，清理一段时间以前回收的文件  
# ---------------------------------------------  
# 定义清理旧文件的函数  
function clearOldFile()  
{  
    echo "delete old file, please wait ..."  

    now=`date "+%s"`  
    for folder in `ls $trashPath`  
    do  
        folderTime=${folder##*_}  # 获取文件名中“_”后的数字  
        gap=`expr $now - $folderTime`  
        # echo $folder    $gap  
        if [ $gap -gt $period ]; then  
            # echo rm $trashPath/$folder  
            /bin/rm -rf $trashPath/$folder  # 删除$period之前的文件夹  
        fi  
    done  
}  
# ---------------------------------------------  
# 设置标志文件路径  
flagFile=$trashPath/.flag  

today=`date "+%d"`  
if [ -f "$flagFile" ]; then  
    lastTime=`cat $flagFile`  # 读取标志文件中的日期  
    if [ "$lastTime" -ne "$today" ]; then  # 如果不是今天的日期就清理  
        clearOldFile  
    fi  
else  # 标志文件不存在也要清理  
    clearOldFile  
fi  
echo $today > $flagFile  # 输出今天的日期到标志文件  
# =====================================================

最后，将alias命令写入.bashrc或者.zshrc中，已保证每次使用bash或者zsh运行rm命令时，都是执行的这个脚本。 

alias rm=$HOME/.rm.sh

使用/bin/rm时，请一定要谨慎。
